### Software review and usage of AI tools

Authors submitting new software for [peer review](/software-review/) are now required to explain potential usage of generative AI tools in their package development. All submission templates now include a mandatory check-box:

    - [ ] Generative AI tools were used to produce some of the material in this submission.

    If so, please describe usage, and include links to any relevant aspects of your repository.

This is the start of our updates to accommodate generative AI tools in package development, as described in our [recent blog post](/blog/2026/02/26/ropensci-ai-policy/). The next phase will involve updates to our [*Dev Guide*](https://devguide.ropensci.org), explaining requirements and recommendations for authors, reviewers, and editors. All updates are intended to permit generative AI tools to be used in any useful way, while minimising the burden on those who volunteer their own time to keep our software peer review service running.

### Software review bot updates

The `ropensci-review-bot` now delivers an initial report to all new software pre-submissions and submissions, identifying the five most similar packages from both all rOpenSci packages, and all CRAN packages. The matches are generated by our [ropensci-review-tools/pkgmatch package](https://docs.ropensci.org/pkgmatch) (itself reviewed in [this review issue](https://github.com/ropensci/software-review/issues/671)). Matching is based on an ["term frequency-inverse document frequency" algorithm](https://en.wikipedia.org/wiki/Tf%E2%80%93idf), using inverse document frequencies from all rOpenSci and CRAN packages. Similar package reports can also be manually triggered (by editors only) with `@ropensci-review-bot similar packages`, like in [this example for the pkgmatch package itself](https://github.com/ropensci/software-review/issues/671#issuecomment-4117805740).

### How to browse the R mailing lists

The [official mailing lists of the R project](https://www.r-project.org/mail.html) like [R-package-devel](https://blog.r-hub.io/2019/04/11/r-package-devel/) are full of important and useful information. How to browse them, given that the default website is not easy to search? You can use the [mail-archive](https://mail-archive.com/r-devel@r-project.org/) website (thanks to Hugo Gruson for the reminder!) or a new project by James Balamuta: the [R Mailing Lists Archive](https://r-mailing-lists.thecoatlessprofessor.com/)!

### "Claude Code: Setting up ast-grep with R support"

Thanks to Mauro Lepore for sharing this blog post by Emil Hvitfeldt: ["Claude Code: Setting up ast-grep with R support"](https://emilhvitfeldt.com/post/ast-grep-r-claude-code/). ast-grep is a tool for querying code by syntax rather than brittle regular expressions. The blog post describes how to add R support to this tool, and how to take advantage of it when using Claude.

### On muffling messages from packages

A follow-up on our post ["Please Shut Up! Verbosity Control in Packages"](/blog/2024/02/06/verbosity-control-packages/).

-   With the {cli} R package you can change the default handler for messages. See the [docs](https://cli.r-lib.org/articles/semantic-cli.html#cli-messages). It seems mostly used to muffle messages, e.g. in [flir](https://github.com/etiennebacher/flir/blob/9254cd01d258d0bafcee41a44e5caa7104fed832/R/lint.R#L104).
