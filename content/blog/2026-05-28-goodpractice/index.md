The [goodpractice package](https://docs.ropensci.org/goodpractice) has been recommended by rOpenSci since it was first started just over 10 years ago by [Gabor Csardi](https://gaborcsardi.org/).
We used to ask our editors to manually run goodpractice on all packages submitted to software peer-review, and then to ask authors to fix any notable issues flagged by the package.
It is now integrated within our own [pkgcheck system](https://docs.ropensci.org/pkgcheck), and used to automatically identify any goodpractice issues with all new submissions.
The package has transitioned maintenance [several times](/blog/2024-06-21-news-june-2024/index.md) before Gabor gave the green light for us to take over maintenance of the package [exactly two years ago (28th May 2024)](https://github.com/ropensci-review-tools/goodpractice/commit/c3f0b8e4c8e2cd7d88361bb670dd79284e6494d1).

We are pleased to announce that we have recently implemented a host of updates and extensions to the package.
These make it both easier to use, and more powerful.

## Easier control of checks

The goodpractice package is a convenience wrapper around several other packages including [rcmdcheck](https://rcmdcheck.r-lib.org), [lintr](https://lintr.r-lib.org), [cyclocomp](https://github.com/gaborcsardi/cyclocomp), and [desc](https://desc.r-lib.org/), along with a few hand-coded checks within the package itself.
The previous version of goodpractice had a total of 230 checks, generally prefixed with the name of the package which defined each check.
Checks were identifiable with the single function, [`all_checks()`](https://docs.ropensci.org/goodpractice/reference/all_checks.html).
The only way to control which checks were run was to pass a `checks` parameter to the main function as a character vector of the names of check you wanted to run.
So, for example, if you wanted to skip the [rcmdcheck](https://rcmdcheck.r-lib.org) checks, you had to do something like this:
``` r
mychecks <- all_checks()[which(!grepl("^rcmdcheck", all_checks()))]
gp(checks=mychecks)
```
That wasn't easy.
But this update finally makes controlling checks much easier, by defining them in "groups" corresponding to the separate packages.
The groups can be seen with the new [`all_check_groups()` function](https://docs.ropensci.org/goodpractice/reference/all_check_groups.html), and controlled with the [`checks_by_group()` function](https://docs.ropensci.org/goodpractice/reference/checks_by_group.html).
This allows checks to be controlled by simply naming the groups of checks you want to run, like this:
``` r
gp(checks=checks_by_group("description", "lintr"))
```

## New check groups

Looking at the documentation for the [`all_check_groups()` function](https://docs.ropensci.org/goodpractice/reference/all_check_groups.html) also shows that this update adds several new groups, including:
- A "description" group to check common issues with `DESCRIPTIION` files, including formatting issues with URLs, DOIs, package names, and author and contributor roles, along with other common `DESCRIPTION` issues.
- An "rd" group to check whether or not `man/*.Rd` function documentation includes both example code and return values (regardless of whether or not documentation files are generated by [roxygen2](https://roxygen2.r-lib.org/)).
- A [roxygen2](https://roxygen2.r-lib.org/) group of checks applied where that package is used to generate documentation.
- An optional [revdep](https://github.com/r-lib/revdepcheck) group for reverse-dependency checking (not run by default).
- "code_structure" checks for a few common issues like functions which are duplicated, or not actually used.
- "package_structure" checks like whether a package has a README, a NEWS file, or whether all files use a `.R` extension, and not `.r`.
- [spelling](https://docs.ropensci.org/spelling/) checks.
- A group of optional [tidyverse](https://tidyverse.org) checks, to check compliance with the [*Tidyverse style guide*](https://style.tidyverse.org/). These are mostly [lintr](https://lintr.r-lib.org/) checks, and are not run by default.
- [urlchecker](https://urlchecker.r-lib.org/) to check whether all URLs are valid.
- A "vignette" group to check that vignette code does not use either `rm()` or `setwd()`.

Many of these checks are powered through [treesitter](https://davisvaughan.github.io/r-tree-sitter/), the extremely efficient syntax parsing library used by GitHub.
The number of checks has been increased from 230 to 338, as listed in the output of the [`all_checks()` function](https://docs.ropensci.org/goodpractice/reference/all_checks.html).

## Improved check reporting

The goodpractice package has always been designed for console output.
This updated version now provides immediate and consistent detail on check progress while running.
The [cli package](https://cli.r-lib.org) is also used to give consistently formatted output for all check groups.

The large increase in numbers of checks also means that directly printing goodpractice results may fill several screens of output.
This update also adds a `groups` parameter to the [`print()` method](https://docs.ropensci.org/goodpractice/reference/print.goodPractice.html), to enable printing check results only for specified groups.
That way you can run the main [`gp()` function](https://docs.ropensci.org/goodpractice/reference/gp.html) once, then step through each check group by printing results only for that group, fixing those, and then moving on to the next group.
